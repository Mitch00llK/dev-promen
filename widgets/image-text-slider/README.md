# Image Text Slider

A modern, responsive slider widget with image backgrounds and text overlay containers for Elementor.

## Features

- Full-width responsive slider with configurable height
- Content overlay container with customizable position and width
- Support for titles, descriptions, and dual call-to-action buttons
- Swiper-powered navigation with customizable arrows and pagination
- GSAP animations for smooth content transitions
- Comprehensive style controls for all elements
- Responsive behavior with mobile-friendly layouts

## Usage

1. Drag and drop the **Image Text Slider** widget into your Elementor section
2. Add slides with images, titles, descriptions, and buttons
3. Configure slider behavior, including autoplay and transitions
4. Style the content container, text, and buttons as needed
5. Adjust positions and responsive behaviors

## Controls

The widget provides comprehensive controls divided into logical sections:

### Content Controls
- Slide management with show/hide options
- Background image selection with position control
- Title and description text fields with show/hide options
- Button configuration with text and link options

### Slider Controls
- Autoplay settings with speed and pause on hover options
- Transition effects and speed
- Navigation controls for arrows and pagination
- GSAP animation enablement

### Style Controls
- Layout controls for slider height and content positioning
- Content container styling with background, border, and shadow options
- Navigation styling for arrows and pagination
- Button styling with comprehensive state controls (normal/hover)

### Typography Controls
- Font styling for titles, descriptions, and buttons
- Text shadow and margin controls
- Separate typography controls for each button type

## Dependencies

- Elementor Pro
- Swiper.js
- GSAP (for animations)

## Development

The widget follows a modular architecture with:
- Separated control sections in the `/controls` directory
- Template rendering in the `/templates` directory
- CSS and JS assets in the `/assets` directory
